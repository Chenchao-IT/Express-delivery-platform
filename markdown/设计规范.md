## 2. 字体系统

### 2.1 字体族

#### 主字体栈
```css
font-family: 'Outfit', -apple-system, BlinkMacSystemFont, 'Segoe UI', 
  'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', 
  Helvetica, Arial, sans-serif;
```

#### 字体说明
```
主字体: "Outfit" (优先使用)
系统字体: -apple-system, BlinkMacSystemFont (macOS/iOS)
中文字体: "PingFang SC" (macOS), "Hiragino Sans GB" (旧版 macOS), 
          "Microsoft YaHei" (Windows)
备用字体: "Helvetica Neue", Helvetica, Arial, sans-serif
```

#### 字体加载策略

**⚠️ 性能关键：必须使用 Next.js Font Optimization**

使用 `next/font/google` 可以：
- ✅ 避免 FOIT (Flash of Invisible Text)
- ✅ 避免 CLS (Cumulative Layout Shift)
- ✅ 自动优化字体文件大小
- ✅ 零运行时开销

**完整实现示例**

```typescript
// app/layout.tsx
import { Outfit } from 'next/font/google'
import type { Metadata } from 'next'

const outfit = Outfit({
  subsets: ['latin'],
  variable: '--font-outfit', // 绑定 CSS 变量
  display: 'swap', // 字体加载时显示系统字体，避免 FOIT
  // ✅ 优化：Outfit 支持 Variable Font，只需指定范围即可
  // 比加载多个字重文件性能更好，且无布局偏移
  weight: ['100', '200', '300', '400', '500', '600', '700', '800', '900'], // 或使用 'variable'
  preload: true, // 预加载字体
  adjustFontFallback: true, // 自动调整备用字体度量
})

export const metadata: Metadata = {
  title: 'Askifier',
  description: '...',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="zh" className={outfit.variable}>
      <body className="font-sans">{children}</body>
    </html>
  )
}
```

```css
/* tailwind.config.ts 或 globals.css */
@layer base {
  :root {
    --font-outfit: 'Outfit', -apple-system, BlinkMacSystemFont, 'Segoe UI', 
      'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif;
  }
  
  body {
    font-family: var(--font-outfit);
  }
}
```

**性能指标目标**
```
FCP (First Contentful Paint): < 1.8s
LCP (Largest Contentful Paint): < 2.5s
CLS (Cumulative Layout Shift): < 0.1
```

### 4.1 通用样式

### 4.1.1 图标与文字对齐规范

#### 对齐原则
```
布局方式: 必须使用 Flexbox (display: flex)
对齐方式: align-items: center (垂直居中)
间距控制: 使用 gap 属性，而非 margin/padding
图标位置: 图标视觉重心比文字基线略低 1px (可选优化)
```

#### 实现示例
```css
/* 左图标右文字 */
.button-with-icon-left {
  display: flex;
  align-items: center;
  gap: 8px; /* 使用 gap 而非 margin */
}

.button-with-icon-left svg {
  width: 20px;
  height: 20px;
  flex-shrink: 0; /* 防止图标被压缩 */
  /* 可选: 微调图标位置，使其视觉居中 */
  position: relative;
  top: 0.5px; /* 向下偏移 0.5px，补偿视觉重心 */
}

/* 左文字右图标 */
.button-with-icon-right {
  display: flex;
  align-items: center;
  gap: 8px;
  flex-direction: row-reverse; /* 或保持顺序，调整 padding */
}

/* 纯图标按钮 */
.button-icon-only {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 40px;
  height: 40px;
}
```

#### 图标尺寸与行高关系
```
字体大小 14px, 行高 20px → 图标 16px
字体大小 16px, 行高 24px → 图标 20px
字体大小 18px, 行高 24px → 图标 20px
字体大小 20px, 行高 24px → 图标 24px

原则: 图标高度 ≈ 行高的 80-90%，确保视觉平衡
```

### 4.8 按钮 Loading 状态规范

**⚠️ 工程关键：防止布局抖动**

当按钮进入 Loading 状态时，必须保持按钮宽度固定，避免布局跳动。

#### 实现方案

**方案 A：固定宽度（推荐）**
```css
.button-loading {
  width: /* 锁定为 loading 前的宽度 */;
  min-width: /* 最小宽度 */;
}
```

**方案 B：Spinner 覆盖文字（推荐）**
```css
.button-loading {
  position: relative;
}

.button-loading .button-text {
  opacity: 0; /* 文字透明，但保留空间 */
}

.button-loading .spinner {
  position: absolute;
  left: 50%;
  top: 50%;
  transform: translate(-50%, -50%);
}
```

#### 规范要求
```
✅ Loading 状态下按钮宽度必须固定
✅ Spinner 与文字不能同时显示（避免视觉混乱）
✅ Loading 动画：旋转 360deg, 1s linear infinite
✅ 颜色：Spinner 颜色与按钮文字颜色一致
```

## 5. 间距与布局

### 5.1 间距系统 (Spacing System)

#### 基础间距单位
基于 4px 的间距系统，确保视觉一致性。

```
4px   - 极小间距（图标与文字、紧密元素）
8px   - 小间距（相关元素组内）
12px  - 中小间距（卡片内元素）
16px  - 标准间距（常用间距）
20px  - 中间距（区块内元素）
24px  - 中大间距（区块间元素）
32px  - 大间距（区块间距）
40px  - 超大间距（主要区块间距）
48px  - 特大间距（页面区块间距）
64px  - 极大间距（页面级区块间距）
```

#### 使用场景

**组件内部间距**
- **按钮内边距**: 根据按钮尺寸规范
- **卡片内边距**: 16px / 20px / 24px（根据卡片大小）
- **表单元素间距**: 12px（标签与输入框）、16px（输入框之间）

**区块间距**
- **Section 间距**: 48px / 64px / 80px（根据页面层级）
- **卡片间距**: 16px / 24px / 32px（根据布局密度）
- **列表项间距**: 12px / 16px / 20px

### 5.2 布局系统 (Layout System)

#### 容器宽度 (Container Width)
```
移动端 (Mobile): 100% - 32px (左右各 16px padding)
平板 (Tablet): 最大 768px
桌面 (Desktop): 最大 1280px (7xl)
大屏 (Large): 最大 1536px
```

#### 栅格系统 (Grid System)

**12列栅格**
- **桌面端**: 12列，gutter 24px
- **平板端**: 8列，gutter 16px
- **移动端**: 4列，gutter 12px

**常用布局模式**

两列布局
```
Desktop: 6:6 (50%:50%)
Tablet: 6:6 (50%:50%)
Mobile: 12:12 (100%:100%, 垂直堆叠)
```

三列布局
```
Desktop: 4:4:4 (33.3%:33.3%:33.3%)
Tablet: 6:6:6 (50%:50%:50%, 第三列换行)
Mobile: 12:12:12 (100%:100%:100%, 垂直堆叠)
```

四列布局
```
Desktop: 3:3:3:3 (25%:25%:25%:25%)
Tablet: 6:6:6:6 (50%:50%:50%:50%, 两行)
Mobile: 12:12:12:12 (100%:100%:100%:100%, 垂直堆叠)
```

#### 响应式断点 (Breakpoints)
```
sm:  640px   - 小屏设备（大手机）
md:  768px   - 平板设备
lg:  1024px  - 小桌面
xl:  1280px  - 桌面
2xl: 1536px  - 大桌面
```

### 5.3 页面布局模式

#### 全宽布局 (Full Width)
- 用于 Hero Section、CTA Section
- 无最大宽度限制，背景可延伸至屏幕边缘

#### 容器布局 (Container Layout)
- 用于内容区域
- 最大宽度限制，居中对齐
- 左右 padding: 16px (移动) / 24px (桌面)

#### 侧边栏布局 (Sidebar Layout)
- 用于博客详情、案例详情等
- 主内容区 + 侧边栏（可选）
- 主内容: 8列，侧边栏: 4列

### 5.4 对齐方式 (Alignment)

#### 文本对齐
- **左对齐**: 正文、列表、表单标签（默认）
- **居中**: 标题、CTA按钮、Hero区域

### 5.5 层级与深度 (Z-Index)

#### 语义化 Z-Index 规范

**⚠️ 工程最佳实践：使用语义化命名而非数字**

在复杂 SaaS 界面中，第三方插件（如 Intercom、Chat widget）或 UI 库（如 Radix UI）可能使用 `z-[50]` 甚至 `z-[9999]`。使用语义化命名可以：
- ✅ 避免层级冲突
- ✅ 提高代码可读性
- ✅ 便于维护和调试

#### Tailwind 配置

```javascript
// tailwind.config.ts
module.exports = {
  theme: {
    extend: {
      zIndex: {
        'base': '0',           // 默认层级
        'dropdown': '10',       // 下拉菜单
        'sticky': '20',         // 固定导航栏 (sticky header)
        'modal-backdrop': '30', // 模态框遮罩层
        'modal': '40',         // 模态框内容
        'toast': '50',         // 提示框 (Toast)
        'cookie': '60',        // Cookie 同意横幅
        'tooltip': '70',       // 工具提示 (通常需要比 Modal 更高)
        'popover': '80',       // 弹出层
        'max': '9999',         // 极端情况（第三方插件）
      }
    }
  }
}
```

#### 使用示例

```tsx
// ✅ 推荐：使用语义化类名
<div className="z-dropdown">下拉菜单</div>
<div className="z-modal-backdrop">遮罩层</div>
<div className="z-modal">模态框</div>
<div className="z-toast">提示框</div>
<div className="z-tooltip">工具提示</div>

// ❌ 不推荐：直接使用数字
<div className="z-10">下拉菜单</div>
<div className="z-30">遮罩层</div>
```

#### 层级关系图

```
z-max (9999)     ← 第三方插件（Intercom, Chat widget）
z-tooltip (70)   ← 工具提示（需要覆盖 Modal）
z-popover (80)   ← 弹出层
z-cookie (60)    ← Cookie 同意横幅
z-toast (50)     ← 提示框
z-modal (40)     ← 模态框内容
z-modal-backdrop (30) ← 模态框遮罩层
z-sticky (20)    ← 固定导航栏
z-dropdown (10)  ← 下拉菜单
z-base (0)       ← 默认层级
```

#### 第三方插件处理

```css
/* 为第三方插件预留最高层级 */

  z-index: 9999 !important; /* 使用 z-max */
}
```

不要直接在组件中使用 Lucide 的 SVG，应封装统一的 `<Icon />` 组件，便于：

```tsx
// components/ui/icon.tsx
import { icons, type LucideIcon } from 'lucide-react';
import { cn } from '@/lib/utils';

interface IconProps {
  name: keyof typeof icons;
  className?: string;
  size?: 12 | 16 | 20 | 24 | 32 | 48;
}

export function Icon({ name, className, size = 20 }: IconProps) {
  const IconComponent = icons[name] as LucideIcon;
  
  return (
    <IconComponent
      className={cn(
        'flex-shrink-0',
        size === 12 && 'w-3 h-3',
        size === 16 && 'w-4 h-4',
        size === 20 && 'w-5 h-5',
        size === 24 && 'w-6 h-6',
        size === 32 && 'w-8 h-8',
        size === 48 && 'w-12 h-12',
        className
      )}
      strokeWidth={2}
    />
  );
}

// 使用示例
<Icon name="User" className="text-text-secondary hover:text-brand" size={20} />
```

### 7.2.1 头像系统 (Avatar System)

**⚠️ SaaS 产品关键组件：用户头像的 Fallback 机制**

SaaS 产品充满用户头像，必须定义清晰的回退策略。

#### 头像尺寸规范
```
小: 32px × 32px (列表项、评论)
中: 48px × 48px (用户卡片、导航栏)
大: 64px × 64px (个人资料页)
超大: 96px × 96px (个人主页)
```

#### Fallback 机制（强制规范）

**方案 A：首字母缩写 + 随机柔和背景色（推荐）**
```
✅ 优点: 让社区看起来更活跃，用户识别度高
实现:
  - 提取用户昵称首字母（中文取第一个字）
  - 根据用户 ID 生成随机柔和背景色（避免过于鲜艳）
  - 文字颜色: 白色 (#FFFFFF)
  - 字体: 14px (小) / 18px (中) / 24px (大) / 32px (超大)
  - 背景色范围: 使用次要调色板（Red, Green, Purple, Cyan, Orange, Yellow）
```

**方案 B：默认灰色人头图标（备选）**
```
⚠️ 不推荐: 视觉单调，社区感弱
实现:
  - 使用 Lucide 的 User 图标
  - 背景: #F4F5F6
  - 图标颜色: #818E9C
```

#### 头像状态指示
```
在线状态: 右下角绿色圆点 (8px × 8px)
离线状态: 无指示
加载中: 使用 Skeleton 组件
加载失败: 自动回退到 Fallback
```

#### 实现示例
```tsx
// components/ui/avatar.tsx
interface AvatarProps {
  src?: string;
  name: string;
  size?: 32 | 48 | 64 | 96;
  showOnline?: boolean;
}

export function Avatar({ src, name, size = 48, showOnline }: AvatarProps) {
  const fallbackColor = generateColorFromName(name); // 根据名称生成颜色
  const initials = getInitials(name); // 提取首字母
  
  return (
    <div className={cn('relative rounded-full', sizeClasses[size])}>
      {src ? (
        <Image src={src} alt={name} className="rounded-full" />
      ) : (
        <div 
          className="rounded-full flex items-center justify-center text-white font-semibold"
          style={{ backgroundColor: fallbackColor }}
        >
          {initials}
        </div>
      )}
      {showOnline && <OnlineIndicator />}
    </div>
  );
}
```

## 8. 交互与动画

### 8.1 动画原则

#### 动画时长
```
微交互: 100-200ms (按钮点击、状态切换)
标准交互: 200-300ms (悬停、展开)
复杂动画: 300-500ms (页面过渡、模态框)
```

#### 缓动函数 (Easing)
```
ease-in-out: 标准过渡（默认）
ease-out: 进入动画（元素出现）
ease-in: 退出动画（元素消失）
linear: 加载动画（旋转、进度条）
```

### 8.2 过渡动画

#### 页面过渡
```
淡入淡出: opacity 0 → 1, 300ms ease-in-out
滑动: translateY(20px) → 0, 300ms ease-out
组合: opacity + translateY, 300ms ease-out
```

#### 组件过渡

**卡片悬停**
```
变换: translateY(-2px)
阴影: shadow-card → shadow-card 增强
时长: 200ms ease-out
```

**按钮交互**
```
点击: scale(0.98), 100ms ease-in
悬停: 背景色过渡, 200ms ease-in-out
```

**模态框**
```
进入: opacity 0 → 1, scale(0.95) → 1, 300ms ease-out
退出: opacity 1 → 0, scale(1) → 0.95, 200ms ease-in
遮罩: opacity 0 → 0.5, 300ms ease-in-out
```

### 8.3 加载动画

#### 骨架屏 (Skeleton)
```
背景: #F4F5F6
动画: 脉冲闪烁 (opacity 0.6 → 1), 1.5s ease-in-out infinite
```

#### 加载指示器
```
旋转: 360deg, 1s linear infinite
颜色: #0066DB
尺寸: 20px / 24px / 32px
```

#### 进度条
```
高度: 4px
背景: #F4F5F6
进度: #0066DB
动画: 宽度 0 → 100%, 线性
```

### 8.4 交互动画

#### 滚动动画
```
触发: 元素进入视口
动画: translateY(30px) → 0, opacity 0 → 1
时长: 600ms ease-out
延迟: 可选的交错延迟 (0ms, 100ms, 200ms)
```

#### 列表项动画
```
进入: translateX(-20px) → 0, opacity 0 → 1
时长: 300ms ease-out
交错: 每个项目延迟 50ms
```

#### 下拉菜单
```
进入: translateY(-10px) → 0, opacity 0 → 1
时长: 200ms ease-out
```

### 8.5 微交互

#### 按钮反馈
```
点击: scale(0.95), 100ms
释放: scale(1), 100ms
```

#### 输入框焦点
```
边框: #EDEEEF → #0066DB
外边框: 0 → 3px rgba(0, 102, 219, 0.1)
时长: 200ms ease-in-out
```

#### 切换动画
```
开关: 滑块位置过渡, 200ms ease-in-out
标签页: 下划线位置, 300ms ease-in-out
```

### 8.6 动画性能

#### 性能优化原则
- 使用 `transform` 和 `opacity` 进行动画（GPU 加速）
- 避免动画 `width`、`height`、`top`、`left` 等属性
- 使用 `will-change` 提示浏览器优化
- 限制同时进行的动画数量

#### 推荐属性
```
✅ transform (translate, scale, rotate)
✅ opacity
✅ filter (blur, brightness)
❌ width, height
❌ top, left, right, bottom
❌ margin, padding
```

### 9.6 响应式表格

#### 移动端表格策略

**⚠️ SaaS 产品推荐：首选横向滚动**

对于 SaaS 后台（Dashboard）和数据报表：
- ✅ **横向滚动**：信息密度最高，一屏可看更多数据
- ⚠️ **卡片化**：适合简单数据，但效率较低（一屏仅 2-3 条）
- ⚠️ **堆叠布局**：适合字段少的表单数据

#### 推荐方案：固定表头 + 横向滚动

```tsx
// 移动端表格实现
<div className="overflow-x-auto">
  <table className="min-w-full">
    <thead className="sticky top-0 bg-white dark:bg-gray-900 z-10">
      <tr>
        <th className="px-4 py-2 text-left">列1</th>
        <th className="px-4 py-2 text-left">列2</th>
        {/* 更多列 */}
      </tr>
    </thead>
    <tbody>
      {/* 表格数据 */}
    </tbody>
  </table>
</div>
```

#### 滚动指示器

```css
/* 显示滚动提示 */
.overflow-x-auto {
  position: relative;
}

.overflow-x-auto::after {
  content: '← 滑动查看更多 →';
  position: absolute;
  right: 0;
  top: 50%;
  transform: translateY(-50%);
  background: linear-gradient(to right, transparent, white);
  padding: 8px 16px;
  font-size: 12px;
  color: #586574;
  pointer-events: none;
}

/* 滚动时隐藏提示 */
.overflow-x-auto.scrolled::after {
  display: none;
}
```

#### 三种方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **横向滚动** ⭐ | 数据报表、多列数据 | 信息密度高、效率高 | 需要用户滑动 |
| 卡片化 | 简单数据、字段少 | 移动端友好 | 效率低、一屏数据少 |
| 堆叠布局 | 表单数据、详情页 | 易读 | 占用空间大 |

#### 实现建议

```tsx
// 响应式表格组件
export function ResponsiveTable({ data, columns }) {
  return (
    <div className="block md:hidden">
      {/* 移动端：卡片化 */}
      <div className="space-y-4">
        {data.map((row) => (
          <Card key={row.id}>
            {columns.map((col) => (
              <div key={col.key} className="flex justify-between">
                <span className="text-gray-600">{col.label}</span>
                <span>{row[col.key]}</span>
              </div>
            ))}
          </Card>
        ))}
      </div>
    </div>
    
    <div className="hidden md:block">
      {/* 桌面端：标准表格 */}
      <div className="overflow-x-auto">
        <table className="min-w-full">
          {/* 表格内容 */}
        </table>
      </div>
    </div>
  )
}
```

## 10. 无障碍设计

### 10.1 颜色对比度

#### WCAG 标准
```
AA 级标准:
- 正文文字: 对比度 ≥ 4.5:1
- 大号文字 (18px+): 对比度 ≥ 3:1
- 非文字元素 (图标、按钮): 对比度 ≥ 3:1

AAA 级标准:
- 正文文字: 对比度 ≥ 7:1
- 大号文字: 对比度 ≥ 4.5:1
```

#### 颜色组合检查

**亮色模式（主要模式）**
```
✅ Text1 (#171E26) on White (#FFFFFF): 15.8:1
✅ Text2 (#586574) on White: 6.2:1
✅ 品牌色 (#0066DB) on White: 4.8:1
✅ Success (#47B881) on White: 3.2:1 (大文字) / 需深色文字
✅ Danger (#F64C4C) on White: 3.9:1 (大文字) / 需深色文字
⚠️ Warning (#FFAD0D) on White: 1.9:1 ❌ 不符合标准
   → 解决方案: 仅作为背景色，配合深色文字 (#171E26)
   → 或使用深色警告色 #D97706 (对比度 4.5:1) 作为文字颜色
```

**深色区块（辅助配色）**
```
✅ Text1 (#F9FAFB) on Dark (#0F172A): 15.8:1
✅ Text2 (#D1D5DB) on Dark: 8.2:1
✅ 品牌色 (#3B82F6) on Dark: 4.8:1
✅ Warning (#F59E0B) on Dark: 4.5:1 ✅
```

#### Warning 色使用规范

**禁止使用场景**
```
❌ #FFAD0D 作为文字颜色在白色背景上
❌ #FFAD0D 作为文字颜色在浅色背景上
```

**强制使用规范**
```
✅ #FFAD0D 作为背景色，配合深色文字 (#171E26)
✅ #FFAD0D 作为图标颜色或装饰元素
❌ #FFAD0D 禁止作为文字颜色（对比度不达标）

✅ 所有警告文字必须使用 #D97706（对比度 4.5:1，符合 WCAG AA）
✅ 在深色区块中使用 #F59E0B 作为警告色（深色背景上对比度达标）

原因：在 SaaS 界面中，警告通常以"文字+图标"形式出现，保证文字本身的对比度达标是最通用的解法。
```

**实现示例**
```css
/* ✅ 正确: 警告背景 + 深色文字 */
.warning-bg {
  background-color: #FFAD0D;
  color: #171E26; /* 对比度 4.5:1 ✅ */
}

/* ✅ 正确: 深色警告文字 */
.warning-text {
  color: #D97706; /* 对比度 4.5:1 ✅ */
}

/* ❌ 错误: 浅色警告文字 */
.warning-text-invalid {
  color: #FFAD0D; /* 对比度 1.9:1 ❌ */
  background: #FFFFFF;
}
```

## 13. 表单验证

### 13.1 验证时机

#### "早奖励，晚惩罚"验证模式

**⚠️ 工程最佳实践：优化用户体验的验证策略**

这是业界验证的 UX 模式，能够：
- ✅ 减少用户输入时的干扰
- ✅ 提供即时正反馈
- ✅ 降低用户挫败感

#### 验证触发规则

**初次输入时（无错误状态）**
```
1. 用户开始输入: 不显示错误 ✅
2. 用户离开焦点 (Blur): 验证并显示错误（如有）⚠️
3. 用户提交表单: 验证所有字段并显示错误 ❌
```

**修正错误时（已有错误状态）**
```
1. 用户修改输入: 实时验证（onChange）✅
2. 格式正确: 立即移除错误提示和红框 ✅
3. 格式仍错误: 保持错误提示，但更新错误信息
```

#### 实现示例

```typescript
// React Hook Form + Zod 示例
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

const schema = z.object({
  email: z.string().email('请输入有效的邮箱地址'),
})

export function ContactForm() {
  const {
    register,
    handleSubmit,
    formState: { errors },
    trigger, // 用于手动触发验证
  } = useForm({
    resolver: zodResolver(schema),
    mode: 'onBlur', // 失焦时验证（初次输入）
  })

  // 如果字段已有错误，实时验证
  const handleChange = async (fieldName: string) => {
    if (errors[fieldName]) {
      await trigger(fieldName) // 实时验证，立即移除错误
    }
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input
        {...register('email', {
          onChange: () => handleChange('email'), // 修正时实时验证
        })}
        aria-invalid={errors.email ? 'true' : 'false'}
      />
      {errors.email && (
        <div role="alert" className="error-message">
          {errors.email.message}
        </div>
      )}
    </form>
  )
}
```

#### 验证时机总结

| 场景 | 验证时机 | 用户体验 |
|------|---------|---------|
| 初次输入 | 失焦时 (onBlur) | 不干扰输入 ✅ |
| 修正错误 | 输入时 (onChange) | 即时反馈 ✅ |
| 提交表单 | 提交时 (onSubmit) | 完整验证 ✅ |

#### 错误提示动画

```css
/* 错误提示出现：淡入 + 下滑 */
.error-message {
  animation: slideDown 200ms ease-out, fadeIn 200ms ease-out;
}

@keyframes slideDown {
  from {
    transform: translateY(-4px);
    opacity: 0;
  }
  to {
    transform: translateY(0);
    opacity: 1;
  }
}

/* 错误提示消失：淡出 */
.error-message.removing {
  animation: fadeOut 150ms ease-in;
}
```
